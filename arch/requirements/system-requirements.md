# System Requirements — RVC Control SW

## 추적성 표기

| ID | 종류 | 내용 |
|----|------|------|
| FR | 기능 요구사항 | 사용자 관점 행위 |
| NFR | 비기능 요구사항 | 품질 속성·제약 |
| SR | 시스템 요구(소프트웨어 관점) | 소프트웨어가 노출할 실행 가능 행위/데이터 |

## 기능 요구사항

| ID | 요구사항 |
|----|-----------|
| FR-001 | RVC는 가청소 패턴으로 바닥을 청소한다. 기본 동작은 **전진하며 청소**이다. |
| FR-002 | 전방 장애물 감지 시 **청소를 중단**하고, 좌 또는 우 회피 기동 후 **전진 청소를 재개**한다. |
| FR-003 | 전방·좌측·우측 모두 장애물인 경우 **후진**한 뒤 좌/우 회피를 수행하고 **전진 청소를 재개**한다. |
| FR-004 | 먼지 감지 시 일정 제어 주기 동안 **청소 출력 강화(부스트)** 상태를 유지한다. |
| FR-005 | HW(모터 드라이버 등)의 세부 제어는 추상화되어 있으며 본 릴리스는 **제어 알고리즘과 상태 결정**에 집중한다. |

## 시스템(소프트웨어) 요구사항

| ID | 요구사항 |
|----|-----------|
| SR-TICK | 소프트웨어는 **주기적인 시간틱**마다 센서 스냅샷을 입력 받아 구동·청소 명령을 산출한다. |
| SR-CMD | 출력 명령은 최소한 `{구동명령, 청소모드}`로 표현 가능해야 한다. |
| SR-DETERMINISTIC | 동일 센서 스냅샷 시퀀스에 대해 결정적 로직으로 재현 가능해야 한다(테스트 가능성). |
| SR-SIM | 검증을 위해 시나리오 기반 시뮬레이터 실행 파일 및 자동 시스템 테스트가 제공된다. |

## 비기능 요구사항 (요약)

상세 시나리오는 `arch/requirements/nfr-quality-attributes.md`.

| ID | 요약 |
|----|------|
| NFR-MAINT-001 | SOLID 원칙 준수(설계 문서에 증거). |
| NFR-READ-001 | C++ 초급자가 따라가기 쉬운 단순 문법·명명 규칙. |
| NFR-TEST-001 | GoogleTest 기반 자동 단위·통합 테스트. |
| NFR-TDD-001 | 기능 요구는 테스트와 매핑되어 실패→통과 주기를 가져야 한다. |
| NFR-COV-001 | 라인 커버리지 **≥ 90%** (**구현 파일 기준**: `src/rvc/**`; `apps/rvc_sim` 제외). |
| NFR-CI-001 | GitHub Actions 기본 브랜치 보호 파이프라인으로 빌드·테스트·커버리지 게이트. |
