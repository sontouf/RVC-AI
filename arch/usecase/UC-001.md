# UC-001: Control automatic cleaning session

## Name

**UC-001 — Control automatic cleaning session**  
사용자가 RVC 자동 청소 세션을 시작하고, 운용 중 중지한다.

## Actor

| Role | Actor |
|------|--------|
| Primary | Resident / User (주거 사용자) |
| Secondary | — |

## Pre-Requisites

- RVC가 **사용자 명령**(시작/중지)을 수신할 수 있는 상태다.
- HW 전원·안전 전제는 `system.md`(블랙박스) 수준에서 충족된다고 가정한다.

## Typical Courses of Events

1. 사용자가 **자동 청소 시작**을 요청한다.
2. 시스템이 세션 상태를 **Cleaning(자동 청소)** 으로 전환한다.
3. 시스템이 UC-002~005에 해당하는 **주행·청소 루프**를 수행할 수 있도록 내부 모드를 설정한다.
4. 사용자가 **중지**를 요청할 때까지 시스템은 자동 청소 세션을 유지한다(다른 UC의 정상/예외 흐름 허용).
5. 사용자가 **중지**를 요청하면 시스템이 세션을 **Idle** 로 전환하고, 주행·청소를 **안전하게 정지**한다.

## Alternative Courses of Events

- **A1. 일시정지(향후 확장)**  
  사용자가 일시정지를 지원하는 경우, Cleaning 유지 없이 동작만 멈추고 재개 시 같은 세션으로 복귀할 수 있다(현 `system.md` 필수 범위 아님 — 명세 추가 시 본 절 확장).

## Exceptional Courses of Events

- **E1. 시작 거부**  
  시작 직전 시스템이 **센서/상태 이상**으로 세션 시작이 불안전하다고 판단하면 Idle을 유지하거나 오류를 알린다(세부 정책은 구현/NFR).

- **E2. 비상 중지**  
  사용자가 **즉시 중지**를 요청하면 모든 진행 중 행동을 중단하고 Idle로 전환한다.

- **E3. 세션 중 치명 오류**  
  시스템이 복구 불가 상태로 판단하면 Idle로 전환하고 청소/주행을 정지한다(로그·알림은 구현).
