# System Test Strategy

## 목적

단위 테스트가 모킹하지 않는 **실행 파일 수준**에서 시나리오를 재생하여 FR-001~FR-004와 시뮬레이터 계약(SR-SIM)을 검증한다.

## 구성 요소

| 구성 요소 | 설명 |
|-----------|------|
| `rvc_sim` | JSON 맵을 읽어 틱 루프를 수행하고 마지막에 요약 JSON을 stdout에 출력 |
| `system_tests/run_all.py` | 시나리오 디렉터리 순회, 프로세스 실행, 기대값 검증 |
| `sim/rvc_grid_gui.py` | 사람이 동일 맵을 시각 확인할 수 있는 GUI (별도 프로세스) |

## 합격 기준

1. 모든 시나리오에서 `rvc_sim`가 비정상 종료 코드를 반환하지 않는다.
2. 요약의 `result` 필드가 `ok` 이다.
3. (선택) GUI가 로컬에서 실행되어 사람의 스모크 검증을 지원한다 — 자동화 게이트는 CLI 스위트가 담당.
